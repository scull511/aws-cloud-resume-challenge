Built a serverless CRUD API microservice on AWS for scalable data management. This project uses API Gateway as an interface for API requests. Behind the scenes, API Gateway triggers an AWS Lambda function written in Python to perform Create, Read, Update and Delete operations on data stored in DynamoDB. This serverless architecture ensures cost-efficiency as resources are only provisioned when the API is in use.

To demonstrate functionality, I used Postman for API testing. By constructing and executing CRUD requests through Postman, I verified the API's ability to interact with DynamoDB tables for data manipulation.

As a microservice, it is largely independent of other components in an application. Through loose coupling it can be scaled, maintained and updated based on its own requirements, with minimal impact on an application. It is easily transferable, and could be used on a variety of data such as users, products and orders.

Tools, Languages & Services Used

API Gateway, Lambda, DynamoDB, Postman

  1. 1. Using IAM, created a Lambda execution role for the function, with a policy that granted the function permission to access CloudWatch and DynamoDB - CloudWatch to create log files, and DynamoDB to perform CRUD operations.
  2. 2. Created the Lambda function, written in Python. The function uses Boto3, the AWS SDK for Python, to work with DynamoDB. When the Lambda function is triggered, the event object contains information about the payload, e.g. the key-value to be created in the DynamoDB table; the table name; the operation to be performed.
  3. 3. Created the DynamoDB table to store the data.
  4. 4. Created a REST API using API Gateway.
  5. 5. Added a resource and POST method to the API.
  6. 6. The POST method integrates with the Lambda function.
  7. 7. Created a stage named 'prod' for the API. The invoke URL, containing the stage name and resource name is what is used in Postman for testing, or would be used in a frontend application interface to make the create/read/update/delete calls to the API.
  8. 8. CREATE - Using Postman, made two PUT requests to the invoke URL, to create an item of data, two times.
  9. 9. HTTP status code of 200 indicating a successful request.
  10. 10. Performing a table scan shows the two items.
  11. 11. READ - Using Postman, making a GET request to the invoke URL to read an item of data.
  12. 12. UPDATE - Using Postman, making a PUT request to the invoke URL to update an item of data.
  13. 13. DELETE - Using Postman, making a DELETE request to the invoke URL to delete an item of data.
