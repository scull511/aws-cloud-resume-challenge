CRC summary

  1. I first designed and created the website with HTML, CSS and JavaScript. I then created an Amazon S3 bucket to store the website files and uploaded the files to the bucket.
  2. I used Amazon CloudFront, a content delivery network service. This allows for secure delivery of data over a low-latency HTTPS connnection. When creating the CloudFront distribution, I chose the bucket domain as the origin and used an Origin Access Control policy to grant access to the S3 bucket from the Cloudfront distribution.
  3. With CloudFront, I set an alternate domain name (CNAME) as the domain and then requested an SSL certificate from AWS Certificate Manager. In Amazon Route 53, I created a DNS record for this alternative domain name. The record routes traffic to the CloudFront distribution.
  4. For the website visitor counter, I created a Dynamo DB table to store the number of views.
  5. I created a Lambda function, written using Python, to record each website view. When creating the Lambda, by enabling the function URL you can use that public URL to invoke the Lambda function, rather than using the API Gateway. The function then uses 'get_item' to retrieve the current 'view' value from the database table, increments the value by 1, and then uses 'put_item' to record the updated value to the database table.
  6. To display the view count, I created a JavaScript function which used a fetch request to call the Lambda function - so all of the above steps were performed, and the view count was then displayed on the web page.
