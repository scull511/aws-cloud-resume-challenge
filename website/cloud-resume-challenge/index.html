The Cloud Resume Challenge is a hands-on project designed to help IT professionals showcase their cloud computing skills. By building a functional resume website entirely on cloud platforms, participants gain practical experience with these technologies while creating a unique and interactive resume that demonstrates their abilities to potential employers.

This website you are viewing is my attempt at the Cloud Resume Challenge. In building this, I leveraged various technologies and services, including AWS services such as S3, CloudFront, Lambda and DynamoDB; GitHub for source control and GitHub Actions to create a CI/CD Pipeline; and Terraform for Infrastructure as Code, to deploy resouces more efficiently and reliably.

This hands-on experience not only allowed me to build a unique and interactive online CV but also provided valuable insights into cloud architecture and best practices.

Tools, Languages & Services Used

HTML/CSS/JavaScript, S3, CloudFront, AWS Certificate Manager, Route 53, Lambda, Python, DynamoDB

    Here's an overview of the key areas I explored:
  1. Website Development: I constructed the website's foundation using HTML, CSS, and JavaScript, laying the groundwork for a functional and visually appealing online presence.
  2. Cloud Storage: To ensure secure and scalable file management, I used Amazon S3 for cloud-based storage of the website's assets.
  3. Content Delivery Optimization: For enhanced website performance, I used Amazon CloudFront, a content delivery network that streamlines content delivery, providing a low latency connection. With CloudFront, you can also set an alternate domain name (CNAME) as the domain. In Amazon Route 53, I created a DNS record for this alternative domain name. The record routes traffic to the CloudFront distribution.
  4. Prioritizing Security: I configured CloudFront to use HTTPS for security and used AWS Certificate Manager to obtain an SSL certificate, thereby encrypting data and safeguarding user information.
  5. Tracking Website Traffic: To gain insights into user engagement, I created a DynamoDB table, a highly scalable database solution within the cloud environment. I then developed a Lambda function, written in Python, that tracks website visits and records them in the table.
  6. Data Visualization Integration: To display the visitor count directly on the website, I used a JavaScript function. This function retrieved the view count from the Lambda function and presents it visually on the web page.
  7. Version Control: For efficient code management, I established a GitHub repository, serving as a centralised repository for all website files.
  8. Automated Deployments: Finally, I implemented a CI/CD pipeline through GitHub Actions. This streamlined approach automates website updates whenever code modifications are introduced, ensuring an efficient deployment process.
  9. The Cloud Resume Challenge not only equipped me with valuable technical skills, it also gave me a stronger understanding of the practical applications of cloud technologies. This experience further strengthens my ability to adapt and thrive in dynamic technological environments.
